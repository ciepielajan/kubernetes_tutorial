Definicje w plikach YAML

Obowiązkowe pola w każdym pliku YAML dla kubernetesa to:
  1. apiVersion, containerPort, kind, metadata/name, spec
    Odpowiedź błędna.
  2. apiVersion, kind, metadata/name, spec
    Odpowiedź poprawna!
  3. apiVersion, containerPort, kind, restartPolicy, metadata/name, spec
    Odpowiedź błędna.

'Wykonanie' pliku yaml na klastrze uzyskamy za pomocą:
  1. kubectl apply -f <plik>
    Odpowiedź poprawna!
  2. kubectl exec -f <plik>
    Odpowiedź błędna.
  3. kubectl deploy -f <plik>
    Odpowiedź błędna.
